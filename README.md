# chess-ai — 纯 JavaScript 国际象棋引擎 + 对弈网页

零依赖的国际象棋引擎，从走法生成到 alpha-beta 搜索全部手写，附带一个单文件网页 UI，可以直接在浏览器里和引擎对弈。

## 快速验收

```bash
# 1. 走法生成正确性：6 个标准 perft 基准局面，节点数逐位精确匹配（约 2 秒）
node test/perft.test.js

# 2. 搜索冒烟测试：找将杀、不送子、搜索不污染局面
node test/search.test.js

# 3. 对弈：起个静态服务打开 index.html 即可下棋
python3 -m http.server 8000   # 然后访问 http://localhost:8000
```

perft 是国际象棋引擎的黄金标准测试：从给定局面枚举所有合法着法序列到指定深度，
节点总数必须与社区公认的精确值完全一致——王车易位、吃过路兵、升变、将军规则里
任何一个细节写错，数字都对不上。本引擎 6 个基准局面（含地狱难度的 Kiwipete）
全部精确通过，初始局面 perft(5) = 4,865,609 在 Node 下约 0.4 秒。

## 文件结构

| 文件 | 说明 |
|---|---|
| `engine.js` | 引擎核心，Node / 浏览器双兼容（约 1000 行） |
| `index.html` | 对弈 UI，全部内联，通过 Web Worker 调引擎（失败时降级主线程） |
| `test/perft.test.js` | perft 正确性测试 |
| `test/search.test.js` | 搜索行为测试 |
| `SPEC.md` | 项目规格（接口定义与验收基准） |

## 引擎技术要点

- **棋盘表示**：0x88 mailbox，越界判断一条位运算搞定
- **走法生成**：完整规则支持——王车易位（含路径受攻击判定）、吃过路兵、四种升变、兵双步
- **搜索**：迭代加深 + alpha-beta（negamax）+ 静态搜索（quiescence，含被将军时的全逃将扩展）
- **置换表**：53 位 Zobrist 哈希（双 31 位随机数组合，降低碰撞），将杀分数按 ply 归一化存取
- **着法排序**：置换表着法 → MVV-LVA 吃子 → 杀手着法
- **评估**：子力价值 + 每兵种 piece-square table
- **规则判定**：将杀 / 逼和 / 五十步（将杀优先）/ 三次重复（幻影过路兵格已规范化，不干扰重复判定）/ 子力不足

## 开发过程

本项目由多智能体流水线协作完成：Sonnet 并行实现引擎与 UI（UI 仅依赖接口规格盲写），
Haiku 独立验证测试，Opus 双视角审计（引擎正确性 + UI 健壮性）共发现 11 个 perft
测不出来的问题（静态搜索将军盲区、置换表 mate 分污染、悔棋软锁死等），全部修复并有回归验证。
